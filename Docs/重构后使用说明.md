# CodexLight 重构后使用说明

## 通信方式

固件支持三种模式：

- `AUTO`：同时监听 USB 串口和 UDP，优先使用最近收到心跳的有线连接。
- `WIRED`：只使用 USB 串口。
- `WIRELESS`：只使用同一局域网内的 UDP。

默认模式在 `Firmware/include/config.h` 的 `DEFAULT_TRANSPORT_MODE` 中设置。

运行时可通过串口切换并保存：

```text
MODE AUTO
MODE WIRED
MODE WIRELESS
STATUS
```

## 手机 AP 配网

没有可用 Wi-Fi 凭据时，ESP32 会开启：

```text
热点：CodexLight-XXXX
密码：123456789
地址：http://192.168.4.1
```

WiFiManager 采用非阻塞模式，AP 配网期间串口仍然可用。

串口命令：

```text
WIFI_CONFIG   # 强制打开配网页
CLEAR_WIFI    # 清除 Wi-Fi 并打开配网页
```

## LED 行为

- 没有收到有效电脑端心跳：GPIO5 黄灯按 1 秒周期闪烁。
- 首次收到有效有线或无线心跳：GPIO6 绿灯闪烁 2 秒。
- 连接动画结束后：显示电脑端发送的 `GREEN`、`RED` 或 `YELLOW` 真实状态。
- 超过 6 秒没有收到当前通信方式的心跳：回到 GPIO5 黄灯闪烁。

## 电脑端 Bridge

有线：

```powershell
python Bridge\codex_light_monitor.py --serial auto --baud 115200
```

无线：

```powershell
python Bridge\codex_light_monitor.py --udp --udp-port 4210
```

同时启用：

```powershell
python Bridge\codex_light_monitor.py --serial auto --baud 115200 --udp --udp-port 4210
```

Bridge 每 2 秒通过已启用的通信方式重发当前状态，作为连接心跳。
